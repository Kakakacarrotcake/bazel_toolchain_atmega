  1. Introduction

    This is a general update to the Microchip MPLAB® XC Network Server. This version updates RLM server to 15.1(BL2) .

    1. Previous Versions

      The previous MPLAB XC Network Server was version 3.10 which was based on RLM 15.0(BL1) .

    2. System Requirements

      MPLAB XC Network Server is available for a variety of 64-bit operating systems, including Microsoft® Windows® 10 and above; Ubuntu® 18.04 and above; or Mac OS® X 13.2 and above.

      The server may also run on various other operating systems including Windows Server, Linux distributions, such as Oracle® Enterprise Linux® and Red Hat® Enterprise Linux as well as older versions of supported operating systems. However, XC License server is not evaluated on these operating systems.

      MPLAB XC Network License Server can be run on Virtual Machines of the supported OS using a virtual machine license for network licenses (SW006021-VM). 32-bit versions of MPLAB XC Network Server are discontinued starting from version 3.00.

    3. Installation

      Run the .exe (Windows), .run (Linux) or .app (OS X) server installer application (for example, XCNetLicServer-3.20-windows.exe) and follow the directions on the screen. Using the default installation directory is recommended.

      If you are using Linux, you must install the compiler using a terminal and from a root account. Install using a Mac OS X account with administrator privileges.

      Full information on installation and operation of the MPLAB XC License Server is covered in the newest version of the XC License Server Manual (DS5002334).

  2. What’s New

    1. Version 3.20

      • Updated to the latest version of RLM - MPLAB XC Network Server v3.20 is based on the Reprise License Manager version 15.1(BL2). This upgrade provides various enhancements and bug fixes made by Reprise to RLM license server.

      • RLM server now requires users to log in for additional security.

          A default rlm.pw file is created with an admin user : The login capability is provided via the RLM password file, named rlm.pw. Beginning in RLM v15.1, if this file doesn’t exist, RLM will create it automatically at startup with a default 'admin:admin:all' account. Reprise Software recommends that you protect access to this file so that ordinary users can't write it. The RLM password file, as well as the directory which contains it, must be read-write to the RLM process. The RLM password file has one line for each user, formatted as follows: 'username:password:list-of-permissions'. The username must not contain a ':' character. If the password field is blank, then the user can log in without supplying a password. When adding new users, you must set a password within 10 minutes or the server will automatically shut down if it is running. To change their password, select the "Change Password" menu item once logged in as that user. The password field is an encrypted hash of the actual password (similar to the Unix password file).

          Adding New Users: Beginning in RLM v15.1, a default user with the username admin and password admin is created if rlm.pw does not already exist.

          To add a new user:

          1. Open rlm.pw (located in the RLM binary directory)
          2. Add a new user with a blank password and desired permissions: johndoe::status
          3. Log in to the web server as this new user and click Change Password to set a password for the user.

          Note: If there are any users without passwords and/or the admin password is still the default, for longer than 10 minutes the server will shut down automatically.

      • RLM web server now supports HTTPS.

          To enable HTTPS in the web server you should use the two new startup options, -sslcert and -sslpriv to point to your SSL certificate and SSL private key. E.g.:

            % ./rlm -sslcert /path/to/cert.pem -sslpriv /path/to/privkey.pem

          Now you can access your web server by typing https://your-server-host:5054 (be sure to include the https:// if you are not automatically redirecting the url via other means). If you are running your server as root/administrator, it is possible to set your web server to run on port 443 (the default HTTPS port) by running the following:

            % ./rlm -sslcert /path/to/cert.pem -sslpriv /path/to/privkey.pem -ws 443

          While running on port 443 and pointing at valid SSL certificate and private key, your browser should automatically redirect to https:// without needing to type it explicitly. i.e., typing yourserver-host/ should take you to https://your-server-host:443.

          Note: If you prefer to generate your own self-signed certificates, your traffic to the web server will still be encrypted, but your browser will likely display a warning before you can get to the web server. Self-signed certificates cannot be generated with a password to use the web server.

      • Running RLM server as root/administrator is no longer restricted

      • Passwords are now case-sensitive. The rlm.pw file will need to be regenerated


        NOTE: For more detailed information about the above-mentioned new features in RLM v15.1, please refer to the RLM License Administration manual.

    2. Version 3.10

      • Updated to the latest version of RLM - MPLAB XC Network Server v3.10 is based on the Reprise License Manager version 15.0(BL1). This upgrade provides various enhancements and bug fixes made by Reprise to RLM license server.

    3. Version 3.00

      • Updated to the latest version of RLM - MPLAB XC Network Server v3.00 is based on the Reprise License Manager version 14.1(BL3). This upgrade provides various enhancements and bug fixes made by Reprise to RLM license server.

        • Updated XC Network Server manual - This release includes an updated manual

      • RLM as windows service can longer be added from web interface - Refer the updated user manual to know how RLM can be added as a Windows service from the command prompt.

    4. Version 2.00

      • Separate Installer - Previous versions of the MPLAB XC Network Server were installed by selecting the Server Install in one of the MPLAB XC Compiler installers. Starting with version 2.00, the Network Server has its own installer.

        • Updated to the newest version of RLM - MPLAB XC Network Server is based on the Reprise License Manager version 11.2(BL2).

      • Status messages include additional information - The status option for the included xclm command now displays additional information on when the HPA (High Priority Access) for the license expires.

      • New XC Network Server manual - This release includes a new manual that includes information on Installation and Operation of the MPLAB XC Network Server. Also included is a “How To” section that answers commonly asked questions.

      • Warnings are issued when HPA and demo licenses are about to expire - A warning is displayed when the license is within 14 days of the HPA or the demo license expiring.

  3. Migration Issues

    1. Before migrating from an older version to a newer version

      • Before taking down the older server, make sure that all the roamed licenses are returned to the server. RLM does not support roaming out licenses from one version and roaming it in on another version

  4. Fixed Issues

    1. Version 3.20

      None

    2. Version 3.10

      None

    3. Version 3.00

      XCLM-140: Network server installer fails on some Windows machines - On some Windows machines, the installer aborts with a message saying C:\ProgramData\Microchip is not present

    4. Version 2.00

      XCLM-77: Unable to Roam a license back into the server - If the Windows version of the rlm command is shut down with a license roamed out, then the license server may forget that the license is roamed out and may issue an error if an attempt to return the license occurs before it times out.

      XCLM-78: Can’t read license data (-102) error issued when running any xlcm command - The Network Server installer that was included with each compiler was not configuring the xclm command. The new, separate installer used for v2.00 of the Network Server now configures the xclm command

  5. Known Issues

    1. Version 3.00

      XCLM-180: Links on the final page of installer do not work - There will be no response on clicking any hyperlinks on the last page of Linux installer.

    2. Version 2.00

      XCLM-56: Host ID reported by 'xclm -hostinfo' changes on a same computer - If a removable network device (like an Android phone or a WiFi adapter) is recognized as the first MAC address found, it will display as the hostID when xclm -hostinfo runs.

      If the device is removed, and that hostID was used to activate a license, then the license will not be valid. xclm will always check each MAC address to see if it matches the hostID in the license file, so any fixed MAC address (with the '-'s removed) can be used as the hostID when the license is activated.
